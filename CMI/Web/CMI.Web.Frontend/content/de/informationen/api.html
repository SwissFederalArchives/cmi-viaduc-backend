SFA Data Search API

The Swiss Federal Archives provide an Application Programming Interface (API) to the metadata on recherche.bar.admin.ch. With this API, developers can create their own applications on top of the SFA's search engine.

All metadata which is publically accessible in the Swiss Federal Archive’s online search is available via the API. Direct access to primary data is not possible, because only allowed for logged users. No login or API key is needed. The API is designed for single searches. It is not designed for bulk downloads. All publically available metadata from the SFA can be conveniently downloaded from the OpenData.swiss platform in one single zipped file (504.0MB).

Technical specification

Host: https://www.recherche.bar.admin.ch/recherche/api/v1/
API type: RESTful.
Exchange format: JSON
Character encoding: UTF-8.

Detailed API documentation in Swagger framework is available here: https://www.recherche.bar.admin.ch/recherche/swagger/index.html?url=/clientdev/swagger/v1/swagger.json

  • Fetch one record using its id: GET https://www.recherche.bar.admin.ch/recherche/api/v1/entities/{id}
  • Fetch multiple records: GET https://www.recherche.bar.admin.ch/recherche/api/v1/entities?ids=1,412,42
  • Search a record: POST request with header: {‘content-type’ : ‘application/json; charset=utf-8'} to https://www.recherche.bar.admin.ch/recherche/api/v1/entities/Search


The API and its documentation are versioned. If you have built your application around version v1, updating your code might be necessary to have it run in version vX.

Example of search code (Python 3)

The following Python 3 code retrieves information on all documents having the word “Feurwehr” or “feurwehr” in any metadata field and prints it to the Python output. Search terms are case-insensitive. 

import json
import urllib.request
HostURL = 'https://www.recherche.bar.admin.ch/recherche/api/v1/entities/Search'
JSON = {"query": { "searchGroups": [{"searchFields": [{"value": "feuerwehr", "key": "allData"}], "fieldOperator": 1}], "groupOperator": 1 }}
params = json.dumps(JSON).encode('UTF-8')
req = urllib.request.Request(HostURL, data=params,headers={'content-type': 'application/json'})
response = urllib.request.urlopen(req)
print(response.read().decode('utf8'))

Examples of JSON search requests (POST)

Searching for a specific phrase “ab cd”

A specific phrase can be searched for with quotes or escaped quotes in the search field value. E.g. "value": '"Leon Schlumpf"', or "value": "\"Leon Schlumpf\"".

N.B.: only search in the title (Titel) and metadata (Metadaten) allow for retrieval of a single term or phrase in the middle of a text, e.g. “Kultur” in “Wissenschaft, Kultur und Lebensbedingungen”. See below (“Searching for a specific reference code”). 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "\"Leon Schlumpf\"",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ], 
        "groupOperator": 1
    }
}

Paging

By default, the query returns only the first 10 results. To get the next results, send the paging parameter to skip the n first results and take m results from that point. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "\"Leon Schlumpf\"",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    },
    "paging": {
        "skip": 100,
        "take": 20,
        "orderBy": "",
        "sortOrder": ""
    }
}

Restricting search to a specific archival level

Restricting search to a specific archival level is achieved by facet filtering. German archival terminology is used:

  • Dokument (document)
  • Dossier (dossier)
  • Serie (series)
  • Teilbestand (sub-fonds)
  • Bestand (fonds)

Here we search the term “feuerwehr” at the document and file levels: 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "feuerwehr",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    },
    "facetsFilters": [
        {
            "filters": [
                "level:\"Dossier\"",
                "level:\"Dokument\""
            ],
            "facet": "level"
        }
    ]
}

Restricting the search to a specific span of years

In the expression creationPeriod.years:[x TO y], x and y can be any years with x<y Here we search the term “feuerwehr” for the creation period from 1950 to 1999: 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "feuerwehr",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    },
    "facetsFilters": [
        {
            "filters": [
                "creationPeriod.years:[1950 TO 1999]"
            ],
            "facet": "creationPeriod.years"
        }
    ]
}

Searching files from a specific fond

Here we search the term “feuerwehr” in all metadata from the fonds "Institut für Politikwissenschaft der Universität Bern (1965-)" : 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "feuerwehr",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    },
    "facetsFilters": [
        {
            "filters": [
                "aggregationFields.bestand:\"Institut für Politikwissenschaft der Universität Bern (1965-)\""
            ],
            "facet": "aggregationFields.bestand"
        }
    ]
}

Searching with Boolean operators


Variant 1: simple search in the same field

Search for all documents containing „loi“ AND “culture” in any field. Variant equivalent to a simple search. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "loi AND culture",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    },
}

Variant 2: Boolean combination of two fields

In a complex search, Boolean conditions are expressed by the fieldOperator.
"fieldOperator": 1 stands for (a AND b)
“fieldOperator”: 2 stands for (a OR b)
“fieldOperator”: 0 stands for NOT(a OR b)

Example: search for all documents containing „loi“ in the titel AND “culture” in any field. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "key": "allData",
                        "value": "loi"
                    },
                    {
                        "key": "title",
                        "value": "culture"
                    }
                ],
            "fieldOperator": 1
        }
        ] 
    }
}

Complex Boolean

The following corresponds to a search for all elements whose title metadata contains “Bundesrat” and “Couchepin”, but whose metadata excludes “Kultur” in all fields. In short, we search:
ALL (title: Bundesrat, title: Couchepin) AND NONE (_all: Kultur),
in other terms:
(title = Bundesrat AND title = Couchepin) AND (NOT anyfield = Kultur) 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "key": "title",
                        "value": "Bundesrat"
                    },
                    {
                        "key": "title",
                        "value": "Couchepin"
                    }
                ],
                "fieldOperator": 1
            },
            {
                "searchFields": [
                    {
                        "key": "allData",
                        "value": "Kultur"
                    }
                ],
                "fieldOperator": 3
            }
        ],
        "groupOperator": 1
    }
}

Searching with wildcards * and ?

Use * or ? as wildcards.
Here we search for all terms beginning with “lo” or with “cult”.
N.B: This does not work in enclosing quotes. Example "lo? OR cult*" works, but "\"lo? OR cult*\"" does not. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "lo? OR cult*",
                        "key": "allData"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    }
}

Searching for a specific partial reference code (Signatur)

Only fields like title (Titel) and metadata (Metadaten) allow for retrieval of a single term or phrase in the middle of a text, e.g. “Kultur” in “Wissenschaft, Kultur und Lebensbedingungen”. This is not the case for alphanumeric identifier fields, notably referenceCode, FormerReferenceCode and Aktenzeichen. For the purpose of indexing optimization, these fields are not tokenized in the search index. This means that if the key is set to referenceCode (signature), a search term such as J2 will not yield any result, as only the exact reference code “J2” would be found. A wildcard, as in “J2*” or “J2?” is necessary to fetch J2.1#1000/1211#1*, J2.1#1000/1211#10*, J2.1#1000/1211#11* etc. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "key": "referenceCode",
                        "value": "J2*"
                    }
                ]
            }
        ]
    }
}

Searching with fuzzy operator ~

The fuzzy operator "bundesr~" also finds Bundesräte, Bundesamt, Bunesrat (wrong spelling) etc. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "key": "title",
                        "value": "bundesr~"
                    }
                ],
                "fieldOperator": 1
            }
        ],
        "groupOperator": 1
    }
}

Proximity searches with the proximity operator ~ n

The proximity operator for n permutations of the word order (displacement, insertion of other words) in the search phrase. With "\"Professor Einstein\" ~2", we fetch “Professor Einstein” but also “Einstein, Professor”. 2 changes have been allowed: 1: swapping the order of both words, 2: inserting a comma. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "value": "\"Professor Einstein\" ~2",
                        "key": "allData"
                    }
                ]
            }
        ]
    }
}

Weighting search terms ^n

The weighting operator n ∈ [1: 100] gives more weight to a specific search term. This influences the order in which the results are displayed. 

{
    "query": {
        "searchGroups": [
            {
                "searchFields": [
                    {
                        "definition": {
                            "key": "title"
                        },
                        "key": "allData",
                        "value": "Bundesrat Couchepin^4"
                    }
                ]
            }
        ]
    }
}
